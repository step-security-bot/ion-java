39  /**
40   * <p>
41   * This implementation differs from the existing non-incremental binary reader implementation in that if
42   * {@link IonReader#next()} returns {@code null} at the top-level, it indicates that there is not (yet) enough data in
43   * the stream to complete a top-level value. The user may wait for more data to become available in the stream and
44   * call {@link IonReader#next()} again to continue reading. Unlike the non-incremental reader, the incremental reader
45   * will never throw an exception due to unexpected EOF during {@code next()}. If, however, {@link IonReader#close()} is
46   * called when an incomplete value is buffered, an {@link IonException} will be raised.
47   * </p>
48   * <p>
49   * Although the incremental binary reader implementation provides performance superior to the non-incremental reader
50   * implementation for both incremental and non-incremental use cases, there is one caveat: the incremental
51   * implementation must be able to buffer an entire top-level value and any preceding system values (Ion version
52   * marker(s) and symbol table(s)) in memory. This means that each value and preceding system values must be no larger
53   * than any of the following:
54   * <ul>
55   * <li>The configured maximum buffer size of the {@link IonBufferConfiguration}.</li>
56   * <li>The memory available to the JVM.</li>
57   * <li>2GB, because the buffer is held in a Java {@code byte[]}, which is indexed by an {@code int}.</li>
58   * </ul>
59   * This will not be a problem for the vast majority of Ion streams, as it is
60   * rare for a single top-level value or symbol table to exceed a few megabytes in size. However, if the size of the
61   * stream's values risk exceeding the available memory, then this implementation must not be used.
62   * </p>
63   * <p>
64   * To enable this implementation, use {@code IonReaderBuilder.withIncrementalReadingEnabled(true)}.
65   * </p>
66   */

70       * Potential future enhancements:
71       * - Split this implementation into a user-level reader and a system-level reader, like the existing implementation.
72       *   This allows this implementation to be used when the user requests a system reader.
73       * - Do not require buffering an entire top-level value. This would be a pretty major overhaul. It may be possible
74       *   to implement using different buffers for each depth. Doing this may also make it possible to avoid buffering
75       *   a value (at any depth) until stepIn() or *Value() is called on it, enabling faster skip-scanning.
76       * - Allow for this implementation to produce the same non-incremental behavior as the old implementation; namely,
77       *   that running out of data during next() would raise an IonException. See the note in the implementation of
78       *   close() below. Implementing this bullet and the previous two bullets would allow us to remove the old binary
79       *   IonReader implementation.
80       * - Add a builder/constructor option that uses a user-provided byte[] directly. This would allow data to be read
81       *   in-place without the need to copy to a separate buffer. Non-incremental behavior (as described in the previous
82       *   bullet) is likely a requirement of this feature.

84       *   resetSymbolTable(), and resetImports().
85       * - When accessed via an iterator, annotations can be parsed incrementally instead of parsing the entire sequence
86       *   up-front.

540         // The system symbol table.
541         private final SymbolTable system = SharedSymbolTable.getSystemSymbolTable(majorVersion);

559         LocalSymbolTableSnapshot() {

564             // It is not expected to grow.
565             listView = new ArrayList<String>(symbols.subList(0, numberOfSymbols));
566             mapView = new HashMap<String, Integer>((int) Math.ceil(numberOfSymbols / 0.75), 0.75f);
567             for (int i = 0; i < numberOfSymbols; i++) {
568                 String symbol = listView.get(i);
569                 if (symbol != null) {
570                     mapView.put(symbol, i);
571                 }
572             }
573             importedTables = new SymbolTable[imports.size()];
574             int i = 0;
575             for (SymbolTable importedTable : imports.values()) {
576                 importedTables[i] = importedTable;

631         @Override
632         public SymbolToken find(String text) {
633             Integer sid = mapView.get(text);
634             if (sid == null) {
635                 return null;
636             }
637             // The following per-call allocation is intentional. When weighed against the alternative of making
638             // 'mapView' a 'Map<String, SymbolToken>` instead of a `Map<String, Integer>`, the following points should
639             // be considered:
640             // 1. A LocalSymbolTableSnapshot is only created when getSymbolTable() is called on the reader. The reader
641             // does not use the LocalSymbolTableSnapshot internally. There are two cases when getSymbolTable() would be
642             // called: a) when the user calls it, which will basically never happen, and b) when the user uses
643             // IonSystem.iterate over the reader, in which case each top-level value holds a reference to the symbol
644             // table that was in scope when it occurred. In case a), in addition to rarely being called at all, it
645             // would be even rarer for a user to use find() to retrieve each symbol (especially more than once) from the
646             // returned symbol table. Case b) may be called more frequently, but it remains equally rare that a user
647             // would retrieve each symbol at least once.
648             // 2. If we make mapView a Map<String, SymbolToken>, then we are guaranteeing that we will allocate at least
649             // one SymbolToken per symbol (because mapView is created in the constructor of LocalSymbolTableSnapshot)
650             // even though it's unlikely most will ever be needed.
651             return new SymbolTokenImpl(text, sid, null);
652         }

768      * Add a shared symbol table import, resolving it from the catalog if possible.
769      * @param name the name of the shared symbol table.
770      * @param version the version of the shared symbol table.
771      * @param maxId the max_id of the shared symbol table. This value takes precedence over the actual max_id for the
772      *              shared symbol table at the requested version.
773      */
774     private void addImport(String name, int version, int maxId) {
775         SymbolTable shared = catalog.getTable(name, version);
776         importMaxId += maxId;
777         if (shared == null) {
778             // No match. All symbol IDs that fall within this shared symbol table's range will have unknown text.
779             imports.put(importMaxId, new SubstituteSymbolTable(name, version, maxId));
780         } else if (shared.getMaxId() != maxId || shared.getVersion() != version) {
781             // Partial match. If the requested max_id exceeds the actual max_id of the resolved shared symbol table,
782             // symbol IDs that exceed the max_id of the resolved shared symbol table will have unknown text.
783             imports.put(importMaxId, new SubstituteSymbolTable(shared, version, maxId));
784         } else {
785             // Exact match; the resolved shared symbol table may be used as-is.
786             imports.put(importMaxId, shared);
787         }
788     }

838     /**
839      * Creates a SymbolToken representation of the given symbol ID.
840      * @param sid a symbol ID.
841      * @return a SymbolToken.
842      */
843     private SymbolToken getSymbolToken(int sid) {
844         if (symbolTokensById == null) {
845             symbolTokensById = new ArrayList<SymbolToken>(symbols.size());
846         }
847         if (symbolTokensById.size() < symbols.size()) {
848             for (int i = symbolTokensById.size(); i < symbols.size(); i++) {
849                 symbolTokensById.add(null);
850             }
851         }
852         if (sid >= symbols.size()) {
853             throw new IonException("Symbol ID exceeds the max ID of the symbol table.");
854         }
855         SymbolToken token = symbolTokensById.get(sid);
856         if (token == null) {
857             String text = symbols.get(sid);
858             ImportLocation importLocation = null;
859             if (text == null) {
860                 // Note: this will never be a system symbol.
861                 if (sid > 0 && sid <= importMaxId) {
862                     importLocation = getImportLocation(sid);
863                 } else {
864                     // All symbols with unknown text in the local symbol range are equivalent to symbol zero.
865                     sid = 0;
866                 }
867             }
868             token = new SymbolTokenImpl(text, sid, importLocation);
869             symbolTokensById.set(sid, token);
870         }
871         return token;
872     }

874     /**
875      * Adds the symbols declared in the given shared symbol table to the reader's current local symbol table, stopping
876      * once the given max_id has been reached.
877      * @param shared a shared symbol table to import.
878      * @param maxId the maximum symbol ID to import from the shared symbol table.
879      */
880     private void addSymbolsFromImport(SymbolTable shared, int maxId) {
881         // The system symbol table is never included in the imports map, so the local SID must be adjusted lower
882         // by the max ID of the system symbol table.
883         int systemMaxId = SharedSymbolTable.getSystemSymbolTable(majorVersion).getMaxId();
884         Integer previousMaxId = lowerKey(maxId);
885         if (previousMaxId == null) {
886             previousMaxId = systemMaxId;
887         }
888         int adjustedMaxId = maxId - previousMaxId;
889         int id = 1;
890         Iterator<String> importedSymbols = shared.iterateDeclaredSymbolNames();
891         while (importedSymbols.hasNext() && id <= adjustedMaxId) {
892             symbols.add(importedSymbols.next());
893             id++;
894         }
895     }

906     /**
907      * Reads a local symbol table from the buffer.
908      * @param marker marker for the start and end positions of the local symbol table in the buffer.
909      */
910     private void readSymbolTable(IonReaderLookaheadBuffer.SymbolTableMarker marker) {
911         peekIndex = marker.startIndex;
912         boolean isAppend = false;
913         boolean hasSeenImports = false;
914         boolean hasSeenSymbols = false;
915         int symbolsPosition = -1;
916         int symbolsEndPosition = -1;
917         while (peekIndex < marker.endIndex) {
918             fieldNameSid = readVarUInt();
919             IonTypeID typeID = readTypeId();
920             calculateEndPosition(typeID);
921             int currentValueEndPosition = valueEndPosition;
922             if (fieldNameSid == SystemSymbolIDs.IMPORTS_ID) {
923                 if (hasSeenImports) {
924                     throw new IonException("Symbol table contained multiple imports fields.");
925                 }
926                 if (typeID.type == IonType.SYMBOL) {
927                     isAppend = readUInt(peekIndex, currentValueEndPosition) == SystemSymbolIDs.ION_SYMBOL_TABLE_ID;
928                     peekIndex = currentValueEndPosition;
929                 } else if (typeID.type == IonType.LIST) {
930                     resetImports();
931                     stepIn();
932                     IonType type = next();
933                     while (type != null) {
934                         String name = null;
935                         int version = -1;
936                         int maxId = -1;
937                         if (type == IonType.STRUCT) {
938                             stepIn();
939                             type = next();
940                             while (type != null) {
941                                 int fieldSid = getFieldId();
942                                 if (fieldSid == SystemSymbolIDs.NAME_ID) {
943                                     if (type == IonType.STRING) {
944                                         name = stringValue();
945                                     }
946                                 } else if (fieldSid == SystemSymbolIDs.VERSION_ID) {
947                                     if (type == IonType.INT) {
948                                         version = intValue();
949                                     }
950                                 } else if (fieldSid == SystemSymbolIDs.MAX_ID_ID) {
951                                     if (type == IonType.INT) {
952                                         maxId = intValue();
953                                     }
954                                 }
955                                 type = next();
956                             }
957                             stepOut();
958                         }
959                         addImport(name, version, maxId);
960                         type = next();
961                     }
962                     stepOut();
963                 }
964                 if (!isAppend) {
965                     // Clear the existing symbols before adding the new imported symbols.
966                     resetSymbolTable();
967                     addSymbolsFromImports();
968                 }
969                 hasSeenImports = true;
970             } else if (fieldNameSid == SystemSymbolIDs.SYMBOLS_ID) {
971                 if (hasSeenSymbols) {
972                     throw new IonException("Symbol table contained multiple symbols fields.");
973                 }
974                 if (typeID.type == IonType.LIST) {
975                     // Just record this position and skip forward. Come back after the imports (if any) are parsed.
976                     symbolsPosition = peekIndex;
977                     symbolsEndPosition = currentValueEndPosition;
978                 }
979                 hasSeenSymbols = true;
980             }
981             peekIndex = currentValueEndPosition;
982         }
983         if (!hasSeenImports) {
984             resetSymbolTable();
985             resetImports();
986         }
987         if (symbolsPosition > -1) {
988             peekIndex = symbolsPosition;
989             valueType = IonType.LIST;
990             valueEndPosition = symbolsEndPosition;
991             stepIn();
992             while (next() != null) {
993                 if (valueType != IonType.STRING) {
994                     symbols.add(null);
995                 } else {
996                     symbols.add(stringValue());
997                 }
998             }
999             stepOut();
1000             peekIndex = valueEndPosition;
1001         }
1002     }

1004     /**
1005      * Advance the reader to the next top-level value. Buffers an entire top-level value, reads any IVMs and/or local
1006      * symbol tables that precede the value, and sets the byte positions of important components of the value.
1007      */
1008     private void nextAtTopLevel() {
1009         if (completeValueBuffered) {
1010             // There is already data buffered, but the user is choosing to skip it.
1011             buffer.seekTo(valueEndPosition);
1012             completeValueBuffered = false;
1013         }
1014         try {
1015             lookahead.fillInput();
1016         } catch (Exception e) {
1017             throw new IonException(e);
1018         }
1019         if (lookahead.moreDataRequired()) {
1020             valueType = null;
1021             valueTypeID = null;
1022             return;
1023         }
1024         completeValueBuffered = true;
1025         if (lookahead.getIvmIndex() > -1) {
1026             peekIndex = lookahead.getIvmIndex();
1027             majorVersion = buffer.peek(peekIndex++);
1028             minorVersion = buffer.peek(peekIndex++);
1029             if (buffer.peek(peekIndex++) != IVM_FINAL_BYTE) {
1030                 throw new IonException("Invalid Ion version marker.");
1031             }
1032             requireSupportedIonVersion();
1033             resetSymbolTable();
1034             resetImports();
1035             lookahead.resetIvmIndex();
1036         }
1037         List<IonReaderLookaheadBuffer.SymbolTableMarker> symbolTableMarkers = lookahead.getSymbolTableMarkers();
1038         if (!symbolTableMarkers.isEmpty()) {
1039             // The cached SymbolTable (if any) is a snapshot in time, so it must be cleared whenever a new symbol
1040             // table is read regardless of whether the new LST is an append or a reset.
1041             cachedReadOnlySymbolTable = null;
1042             for (IonReaderLookaheadBuffer.SymbolTableMarker symbolTableMarker : symbolTableMarkers) {
1043                 readSymbolTable(symbolTableMarker);
1044             }
1045             lookahead.resetSymbolTableMarkers();
1046         }
1047         peekIndex = lookahead.getValueStart();
1048         if (lookahead.getAnnotationSids().isEmpty()) {
1049             valueTypeID = lookahead.getValueTid();
1050             valueType = valueTypeID.type;
1051         } else {
1052             hasAnnotations = true;
1053             valueTypeID = IonTypeID.TYPE_IDS[buffer.peek(peekIndex++)];
1054             int wrappedValueLength = valueTypeID.length;
1055             if (valueTypeID.variableLength) {
1056                 wrappedValueLength = readVarUInt();
1057             }
1058             valueType = valueTypeID.type;
1059             if (valueType == IonTypeID.ION_TYPE_ANNOTATION_WRAPPER) {
1060                 throw new IonException("Nested annotations are invalid.");
1061             }
1062             if (peekIndex + wrappedValueLength != lookahead.getValueEnd()) {
1063                 throw new IonException("Mismatched annotation wrapper length.");
1064             }
1065         }
1066         valueStartPosition = peekIndex;
1067         valueEndPosition = lookahead.getValueEnd();
1068         lookahead.resetNopPadIndex();
1069     }

1109     /**
1110      * Advance the reader to the next value within a container, which must already be buffered.
1111      */
1112     private void nextBelowTopLevel() {
1113         // Seek past the previous value.
1114         if (peekIndex < valueEndPosition) {
1115             peekIndex = valueEndPosition;
1116         }
1117         if (peekIndex >= containerStack.peek().endPosition) {
1118             endContainer();
1119         } else {
1120             if (containerStack.peek().type == IonType.STRUCT) {
1121                 fieldNameSid = readVarUInt();
1122             }
1123             IonTypeID typeID = readTypeId();
1124             while (typeID.isNopPad) {
1125                 calculateEndPosition(typeID);
1126                 peekIndex = valueEndPosition;
1127                 if (peekIndex >= containerStack.peek().endPosition) {
1128                     endContainer();
1129                     return;
1130                 }
1131                 if (containerStack.peek().type == IonType.STRUCT) {
1132                     fieldNameSid = readVarUInt();
1133                 }
1134                 typeID = readTypeId();
1135             }
1136             calculateEndPosition(typeID);
1137             if (valueType == IonTypeID.ION_TYPE_ANNOTATION_WRAPPER) {
1138                 hasAnnotations = true;
1139                 annotationSids.clear();
1140                 annotationsLength = readVarUInt();
1141                 annotationStartPosition = peekIndex;
1142                 peekIndex = annotationStartPosition + annotationsLength;
1143                 typeID = readTypeId();
1144                 if (typeID.isNopPad) {
1145                     throw new IonException(
1146                         "Invalid annotation wrapper: NOP pad may not occur inside an annotation wrapper."
1147                     );
1148                 }
1149                 if (valueType == IonTypeID.ION_TYPE_ANNOTATION_WRAPPER) {
1150                     throw new IonException("Nested annotations are invalid.");
1151                 }
1152                 long annotationWrapperEndPosition = valueEndPosition;
1153                 calculateEndPosition(typeID);
1154                 if (annotationWrapperEndPosition != valueEndPosition) {
1155                     throw new IonException(
1156                         "Invalid annotation wrapper: end of the wrapper did not match end of the value."
1157                     );
1158                 }
1159             } else {
1160                 annotationStartPosition = -1;
1161                 annotationsLength = -1;
1162                 hasAnnotations = false;
1163                 if (valueEndPosition > containerStack.peek().endPosition) {
1164                     throw new IonException("Value overflowed its container.");
1165                 }
1166             }
1167             if (!valueTypeID.isValid) {
1168                 throw new IonException("Invalid type ID.");
1169             }
1170             valueStartPosition = peekIndex;
1171         }
1172     }

1262     @Override
1263     public IntegerSize getIntegerSize() {
1264         if (valueType != IonType.INT || isNullValue()) {
1265             return null;
1266         }
1267         if (valueTypeID.length < INT_SIZE_IN_BYTES) {
1268             // Note: this is conservative. Most integers of size 4 also fit in an int, but since exactly the
1269             // same parsing code is used for ints and longs, there is no point wasting the time to determine the
1270             // smallest possible type.
1271             return IntegerSize.INT;
1272         } else if (valueTypeID.length < LONG_SIZE_IN_BYTES) {
1273             return IntegerSize.LONG;
1274         } else if (valueTypeID.length == LONG_SIZE_IN_BYTES) {
1275             // Because creating BigIntegers is so expensive, it is worth it to look ahead and determine exactly
1276             // which 8-byte integers can fit in a long.
1277             if (valueTypeID.isNegativeInt) {
1278                 // The smallest negative 8-byte integer that can fit in a long is -0x80_00_00_00_00_00_00_00.
1279                 int firstByte = buffer.peek(valueStartPosition);
1280                 if (firstByte < MOST_SIGNIFICANT_BYTE_OF_MIN_LONG) {
1281                     return IntegerSize.LONG;
1282                 } else if (firstByte > MOST_SIGNIFICANT_BYTE_OF_MIN_LONG) {
1283                     return IntegerSize.BIG_INTEGER;
1284                 }
1285                 for (int i = valueStartPosition + 1; i < valueEndPosition; i++) {
1286                     if (0x00 != buffer.peek(i)) {
1287                         return IntegerSize.BIG_INTEGER;
1288                     }
1289                 }
1290             } else {
1291                 // The largest positive 8-byte integer that can fit in a long is 0x7F_FF_FF_FF_FF_FF_FF_FF.
1292                 if (buffer.peek(valueStartPosition) > MOST_SIGNIFICANT_BYTE_OF_MAX_LONG) {
1293                     return IntegerSize.BIG_INTEGER;
1294                 }
1295             }
1296             return IntegerSize.LONG;
1297         }
1298         return IntegerSize.BIG_INTEGER;
1299     }

1663     /**
1664      * Reads a decimal value as a BigDecimal.
1665      * @return the value.
1666      */
1667     private BigDecimal readBigDecimal() {
1668         int length = valueEndPosition - peekIndex;
1669         if (length == 0) {
1670             return BigDecimal.ZERO;
1671         }
1672         int scale = -readVarInt();
1673         BigDecimal value;
1674         if (length < LONG_SIZE_IN_BYTES) {
1675             // No need to allocate a BigInteger to hold the coefficient.
1676             long coefficient = 0;
1677             int sign = 1;
1678             if (peekIndex < valueEndPosition) {
1679                 int firstByte = buffer.peek(peekIndex++);
1680                 sign = (firstByte & HIGHEST_BIT_BITMASK) == 0 ? 1 : -1;
1681                 coefficient = firstByte & LOWER_SEVEN_BITS_BITMASK;
1682             }
1683             while (peekIndex < valueEndPosition) {
1684                 coefficient = (coefficient << VALUE_BITS_PER_UINT_BYTE) | buffer.peek(peekIndex++);
1685             }
1686             value = BigDecimal.valueOf(coefficient * sign, scale);
1687         } else {
1688             // The coefficient may overflow a long, so a BigInteger is required.
1689             value = new BigDecimal(readIntAsBigInteger(valueEndPosition), scale);
1690         }
1691         return value;
1692     }

1695      * Reads a decimal value as a Decimal.
1696      * @return the value.
1697      */
1698     private Decimal readDecimal() {
1699         int length = valueEndPosition - peekIndex;
1700         if (length == 0) {
1701             return Decimal.ZERO;
1702         }
1703         int scale = -readVarInt();
1704         BigInteger coefficient;
1705         length = valueEndPosition - peekIndex;
1706         if (length > 0) {
1707             // NOTE: unfortunately, there is no BigInteger(int signum, byte[] bits, int offset, int length) constructor,
1708             // so copying to scratch space is always required.
1709             byte[] bits = copyBytesToScratch(peekIndex, length);
1710             int signum = getAndClearSignBit(bits);
1711             // NOTE: there is a BigInteger.valueOf(long unscaledValue, int scale) factory method that avoids allocating
1712             // a BigInteger for coefficients that fit in a long. See its use in readBigDecimal() above. Unfortunately,
1713             // it is not possible to use this for Decimal because the necessary BigDecimal constructor is
1714             // package-private. If a compatible BigDecimal constructor is added in a future JDK revision, a
1715             // corresponding factory method should be added to Decimal to enable this optimization.
1716             coefficient = new BigInteger(signum, bits);
1717             if (coefficient.signum() == 0 && signum < 0) {
1718                 return Decimal.negativeZero(scale);
1719             }
1720         }
1721         else {
1722             coefficient = BigInteger.ZERO;
1723         }
1724         return Decimal.valueOf(coefficient, scale);
